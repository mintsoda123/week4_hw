# MIT 딥러닝 입문 - Day 2: Deep Generative Modeling

---

## 주제
- 데이터를 보고 새로운 데이터를 생성하는 생성적 딥러닝 기법 소개
- 지도학습(Supervised Learning)과 비지도학습(Unsupervised Learning)의 차이 이해
- Autoencoder, Variational Autoencoder(VAE), Generative Adversarial Network(GAN)의 원리와 구조 학습
- 생성 모델의 실제 응용 사례(편향 제거, 이상치 탐지, 이미지 합성 등) 탐구

---

## 핵심개념 정리

### 1. 생성 모델(Generative Modeling)이란?

**[1-1] 지도학습 vs 비지도학습**
- 지도학습(Supervised Learning): 데이터를 레이블에 매핑하는 함수를 학습
  - 예: 이미지 → 클래스 레이블
- 비지도학습(Unsupervised Learning): 레이블 없이 데이터의 내재된 구조를 학습
  - 예: 클러스터링, PCA
- 생성 모델은 비지도학습의 일종

**[1-2] 생성 모델의 목표**
- 훈련 분포(training distribution)에서 샘플을 입력받아 그 분포를 표현하는 모델 학습
- 학습한 분포에서 새로운 샘플을 생성 가능

**[1-3] 왜 생성 모델이 필요한가?**
- 편향 제거(Debiasing): 불균형한 데이터셋에서 공정하고 대표성 있는 데이터 생성
  - 예: 특정 집단이 과소 대표된 경우, 생성 모델로 보완 데이터 생성
- 이상치 탐지(Outlier Detection): 자율주행에서 보행자 등 비정상적 상황 감지
- 데이터 증강(Data Augmentation): 훈련 데이터 부족 문제 해결

---

### 2. 잠재 변수 모델(Latent Variable Models)

**[2-1] 잠재 변수의 개념**
- 데이터를 설명하는 숨겨진 요인(hidden factors)
- 고차원 입력 데이터를 저차원의 의미있는 표현으로 압축

**[2-2] 이 강의에서 다루는 두 가지 모델**
1. Autoencoder / Variational Autoencoder (VAE)
2. Generative Adversarial Network (GAN)
- 두 모델 모두 잠재 변수 모델에 해당함

---

### 3. Autoencoder (오토인코더)

**[3-1] 오토인코더의 구조**
- 인코더(Encoder): 고차원 입력 데이터 → 저차원 잠재 공간(latent space)으로 압축
- 잠재 벡터(Latent vector, z): 압축된 표현
- 디코더(Decoder): 잠재 표현 → 원본 데이터 재구성(reconstruction)

**[3-2] 오토인코더의 학습 방법**
- 레이블이 없으므로 입력 데이터 자체를 활용
- 손실 함수: 재구성 손실(Reconstruction Loss)
  - L(x, x̂) = ‖x - x̂‖²  (평균 제곱 오차, MSE)
  - 픽셀 단위로 원본과 재구성 결과의 차이를 최소화
- 레이블 없이도 학습 가능한 자기지도학습(Self-supervised Learning)

**[3-3] 잠재 공간의 크기와 재구성 품질**
- 병목(bottleneck)이 작을수록: 더 많은 압축 → 재구성 품질 저하
- 병목이 클수록: 덜 압축 → 재구성 품질 향상
- 압축 정도와 재구성 품질 사이의 트레이드오프 존재

**[3-4] 오토인코더의 활용: 표현 학습(Representation Learning)**
- 레이블 없이도 데이터의 압축된 표현 학습 가능
- 학습된 잠재 표현을 다운스트림 작업에 활용 가능

**[3-5] 오토인코더의 한계**
- 결정론적(Deterministic): 같은 입력에 항상 같은 출력
- 잠재 공간에서 새로운 샘플을 생성하기 어려움
- 잠재 공간의 구조가 불연속적이고 불완전할 수 있음

---

### 4. Variational Autoencoder (VAE)

**[4-1] VAE의 핵심 아이디어**
- 오토인코더에 확률적(Probabilistic) 요소를 추가
- 결정론적 잠재 벡터 대신 확률 분포를 학습
- 인코더: 잠재 공간의 확률 분포를 추론
- 디코더: 잠재 분포가 주어졌을 때 입력 공간의 확률 분포를 추론

**[4-2] VAE의 학습 과정**
- 인코더와 디코더 모두 확률적(Probabilistic) 성격을 가짐
- 인코더 파라미터(φ)와 디코더 파라미터(θ)를 각각 학습
- 손실 함수 = 재구성 손실 + 정규화 항(Regularization Term)

**[4-3] VAE 손실 함수**
- 재구성 손실: 입력과 재구성 출력 사이의 차이
- 정규화 항: KL-발산(KL-Divergence)
  - D(q_φ(z|x) ‖ p(z))
  - 학습된 분포 q와 사전 분포 p(z) 사이의 거리 측정
  - 일반적으로 p(z) = 평균 0, 분산 1의 정규 가우시안 분포 사용

**[4-4] 정규화의 직관적 이해**
- 정규화 없는 경우:
  - 연속성(Continuity) 미보장: 잠재 공간에서 가까운 두 점이 다르게 디코딩될 수 있음
  - 완전성(Completeness) 미보장: 일부 잠재 공간 영역이 의미없는 출력을 생성
- 정규화 있는 경우:
  - 잠재 공간이 부드럽고 연속적으로 분포
  - 잠재 공간의 임의 지점에서 의미있는 샘플 생성 가능
- 정규화 강도와 재구성 품질 사이의 균형 필요

**[4-5] 재파라미터화 트릭(Reparameterization Trick)**
- 문제: 샘플링 연산은 확률적이므로 역전파(Backpropagation) 불가
- 해결: 확률성을 파라미터에서 분리
  - z = μ + σ × ε  (ε ~ N(0, 1))
  - μ와 σ는 학습 가능한 파라미터, ε은 외부 노이즈
  - 확률성을 ε으로 이전하여 μ, σ에 대한 역전파 가능

**[4-6] VAE로 새 데이터 생성하는 방법**
1. 사전 분포 p(z)에서 잠재 벡터 z를 샘플링
2. 디코더에 z를 입력하여 새로운 데이터 생성
- 연속적인 잠재 공간 덕분에 부드러운 보간(interpolation) 가능

**[4-7] VAE의 활용: 편향 제거(Debiasing)**
- 데이터의 잠재 변수(예: 피부색, 성별)를 학습
- 과소 대표된 그룹에 대해 새로운 데이터 생성
- 더 공정하고 균형잡힌 훈련 데이터셋 구성 가능

### 5. Generative Adversarial Networks (GAN)

**[5-1] GAN의 핵심 아이디어**
- 밀도를 명시적으로 모델링하지 않고 샘플링을 통해 새로운 데이터 생성
- 문제: 복잡한 분포에서 직접 샘플링 불가
- 해결: 단순한 분포(노이즈)에서 샘플링하고 복잡한 분포로 변환하는 함수를 학습

**[5-2] GAN의 두 가지 구성 요소**
- 생성자(Generator):
  - 노이즈(z)를 입력받아 가짜 데이터를 생성
  - 판별자를 속이도록 학습
- 판별자(Discriminator):
  - 실제 데이터와 생성자가 만든 가짜 데이터를 구별
  - 실제 데이터: 1, 가짜 데이터: 0으로 분류하도록 학습

**[5-3] GAN의 학습 과정 (적대적 훈련)**
- 생성자와 판별자가 서로 경쟁하며 동시에 학습
- 판별자 목표: 실제 데이터와 가짜 데이터를 최대한 잘 구별
- 생성자 목표: 판별자가 가짜라고 판단하지 못하도록 실제와 유사한 데이터 생성
- 이상적인 수렴: 생성자가 실제 데이터와 구별 불가능한 데이터를 생성

**[5-4] GAN의 손실 함수**
- 판별자 손실: 실제 데이터는 1, 가짜 데이터는 0으로 분류하도록 최적화
- 생성자 손실: 판별자가 가짜 데이터를 1(실제)로 분류하도록 최적화
- 두 손실 함수가 서로 반대 방향으로 작용하는 min-max 게임 구조

**[5-5] GAN의 이미지 합성 응용**
- 얼굴 생성: 실제와 구별 불가능한 고해상도 얼굴 이미지 생성
- 쌍 변환(Paired Translation):
  - 스케치 → 컬러 이미지
  - 흑백 → 컬러
  - 낮 → 밤 등 조건부 이미지 변환
- 분포 변환(Distribution Transformation):
  - 말 이미지 → 얼룩말 이미지
  - 한 도메인의 이미지를 다른 도메인으로 변환

### 6. 생성 모델의 응용

**[6-1] 이미지 합성**
- 새로운 얼굴, 사물, 장면 등 실제처럼 보이는 이미지 생성
- 영화, 게임, 예술 등 창작 분야 활용

**[6-2] 편향 제거(Debiasing)**
- 데이터셋의 불균형을 생성 모델로 해소
- 더 공정한 AI 모델 훈련을 위한 균형잡힌 데이터 생성

**[6-3] 이상치 탐지(Outlier Detection)**
- 정상 데이터의 분포를 학습한 후 분포에서 벗어난 이상치 탐지
- 자율주행, 의료 영상 등 안전이 중요한 분야에서 활용

**[6-4] 데이터 증강(Data Augmentation)**
- 레이블이 있는 훈련 데이터가 적을 때 생성 모델로 추가 데이터 생성
- 모델의 일반화 성능 향상

---

### 7. 두 모델의 비교

**[7-1] VAE vs GAN**

| 항목 | VAE | GAN |
|------|-----|-----|
| 접근 방식 | 확률 분포 명시적 모델링 | 암묵적 샘플링 |
| 잠재 공간 | 연속적이고 구조적 | 구조가 덜 명확함 |
| 생성 품질 | 다소 흐릿(blurry)할 수 있음 | 매우 선명하고 사실적 |
| 학습 안정성 | 상대적으로 안정적 | 훈련이 불안정할 수 있음 |
| 보간(Interpolation) | 용이 | 어려울 수 있음 |

**[7-2] 공통점**
- 둘 다 잠재 변수 모델
- 둘 다 새로운 데이터 생성 가능
- 둘 다 인코더-디코더 또는 생성자-판별자 구조 활용

---

### 8. 강의 요약

**[8-1] 생성 모델의 패러다임 전환**
- 기존 딥러닝: 데이터로부터 패턴을 학습 (지도학습)
- 생성 모델: 데이터로부터 학습하여 더 많은 데이터를 생성 (새로운 패러다임)

**[8-2] 핵심 모델 정리**
- Autoencoder: 데이터 압축 및 표현 학습, 결정론적
- VAE: 확률적 오토인코더, 연속적 잠재 공간, 새 데이터 생성 가능
- GAN: 적대적 훈련, 매우 사실적인 이미지 생성 가능
